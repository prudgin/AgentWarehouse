## Boundary rule

`docs/planning/future-work.md` is **pre-decision**. `.tickets/` is **post-decision**. When an item in future-work becomes a scoped, ready-to-do unit of work, transition it: file a ticket in `.tickets/` and delete the future-work entry. The transition deletes the future-work line — it does not coexist in both places.

This matches the warehouse convention; see the warehouse `docs/planning/` for the canonical statement of the boundary.
