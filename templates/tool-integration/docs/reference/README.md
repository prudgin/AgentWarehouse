# Reference

How the code works. For tool-integration projects, this directory is often **light or empty** — most procedural knowledge lives in skills (`.claude/skills/`) and most operational knowledge lives in `_tools/` scripts. Use this directory for first-party code that needs documentation beyond what a skill or script header captures.

## Rules

- Update **after** implementing or significantly changing first-party code.
- Link to `glossary.md` and `docs/domain/` for context.
- Link to `docs/adr/` for decisions that shaped the design.
- Describe behaviour, interfaces, and invariants. Don't paste code; link to it.

If this directory ends up empty, that's fine — delete it and remove the reference from CLAUDE.md, or leave it for future first-party code.
