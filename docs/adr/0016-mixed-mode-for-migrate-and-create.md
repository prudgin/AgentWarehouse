# Mixed-mode for `/migrate-project` and `/create-project`

[ADR-0011](0011-interactive-skills-refuse-auto-mode.md) declares that skills which ask the user questions refuse auto mode. Read strictly, that blocked `/migrate-project` and `/create-project` from running AFK. But these two skills are mostly mechanical (file copies, symlinks, doc transfers, placeholder substitution — ~80% of any migration) with a small destructive minority (deleting stale docs, splitting `decisions.md` into ADRs, resolving conflicts) — they aren't pure-interview skills. We move both onto the `/work-issue` and `/finish` pattern: **auto for reversible local actions; pause-and-surface for destructive or shared-state ones**. ADR-0011 is narrowed in scope (not superseded) to *purely interview-shaped* skills — `/grill`, `/intake-target-project`, `/to-issues`, `/triage`, `/improve-codebase-architecture` — where without a human there is genuinely nothing to do. The cost: each mixed-mode skill must enumerate its destructive-op set explicitly in its SKILL.md, and a wrong enumeration risks AFK destruction without confirmation. Mitigation: pause defaults err on the side of confirm — anything that isn't manifestly an additive copy/move/rename pauses.
