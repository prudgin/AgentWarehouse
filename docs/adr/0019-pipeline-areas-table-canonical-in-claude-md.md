# Pipeline-areas table is canonical in CLAUDE.md

The `pipeline` template's CLAUDE.md carries a "Pipeline areas" table listing each stage with its entry point, code dir, and output. The same content is partially restated in `docs/reference/<stage>.md` per-stage docs. We make **CLAUDE.md the canonical source**: the table is the authoritative list of stages, and per-stage reference docs link back to it for "what stages exist". Reasons: (a) CLAUDE.md is the first file an agent reads, and "what stages exist in this pipeline" is doc-map content, not implementation detail — it belongs at the entry; (b) the table includes *Orchestration*, which has no `docs/reference/` doc, so `docs/reference/` cannot be the canonical list of stages without inventing a phantom doc; (c) `/finish` step 4 already validates CLAUDE.md doc-map accuracy, so cross-checking the table fits the existing cleanup ritual. Rejected alternatives: `docs/reference/` canonical (leaves Orchestration homeless and forces a click to learn the pipeline shape), and auto-generation from per-stage frontmatter (over-engineering at this scale, adds a build step the warehouse philosophy has otherwise resisted). Drift mitigation: a one-line note above the table — "If you change this table, run `/finish` — it cross-checks against `docs/reference/`."
