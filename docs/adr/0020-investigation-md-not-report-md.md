# Investigation deliverable is `INVESTIGATION.md`, not `REPORT.md`

The analysis-template's primary deliverable per investigation lives at `analysis/YYYY-MM-DD-<topic>/<file>.md`. Originally the file was called `REPORT.md`. We rename to `INVESTIGATION.md`. Two reasons. **(1) Harness collision.** Claude Code's system rules conflate any `REPORT.md` write with an "agent's findings-to-parent report" pattern and block the `Write` tool, forcing analysts to fall back to Bash heredoc — a permanent friction surfaced by the 2026-05-04 template self-test. **(2) Vocabulary alignment.** The warehouse already uses *investigation* in the analyse-chain doc map ("investigation outputs", "investigation dir", "discovery thread across investigations"); the term is canonical. The filename now agrees with the concept it carries. Cost: one-time mechanical rename across `skills/{start-analysis,finish-analysis,finish}/SKILL.md`, the four templates' `analysis/README.md` and `analysis/analysis-landscape.md`, and the warehouse's own existing `analysis/2026-05-04-template-self-test/REPORT.md`. Trade-off considered: keep `REPORT.md` and document the heredoc workaround. Rejected because the workaround is permanent friction that every analyst will hit, and because the rename eliminates the friction at the root rather than papering over it. *Report* survives in informal speech; the file name just stops colliding with the harness.
