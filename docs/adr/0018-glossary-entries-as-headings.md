# Glossary entries use `### Term` headings, not bold-paragraph labels

[ADR-0002](0002-glossary-keeps-name-adopts-content-contract.md) pinned `glossary.md` and the content contract (canonical term, "Avoid" synonyms, relationships, example dialogue, flagged ambiguities) but did not specify the entry-level markdown shape. Existing entries use a bold-paragraph label: `**Term**:` followed by a definition. That style reads densely (good for scanning) but breaks deep-linking — markdown anchors only resolve to headings, so a link like `glossary.md#fixture` from `docs/reference/ingest.md` does not jump the reader to the **Fixture** entry. We move every glossary entry to `### Term` heading style so anchors resolve. The cost is two more vertical lines per entry; the gain is mechanical deep-linking, which the warehouse increasingly relies on (provenance chains in the `analysis` template, cross-references between `docs/reference/` / `docs/domain/` / `docs/adr/` and the glossary). Rejected alternatives: drop deep-links into the glossary entirely (forces every reader to scroll/search, half-defeats the findings-provenance rule), and `<a id="term">` HTML anchors with bold labels (functional but ugly, and adds a parallel anchor-management surface that has nothing to do with markdown). Migration is mechanical: convert each entry's `**Term**:` opening to a `### Term` heading, then promote the definition to its own paragraph.
