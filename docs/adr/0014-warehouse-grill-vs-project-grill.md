# Warehouse-grill and project-grill are separate skills

The warehouse needs to grill the user about *target projects* it is setting up, not just about its own design. We could give `grill` a target-mode flag (one skill, two modes) or add a second skill (`intake-target-project`) that's distinct from `grill`. We chose two skills: `grill` keeps a single, well-defined responsibility (align on the *current* project and write to its `glossary.md` / `docs/adr/`), while `intake-target-project` is warehouse-only and writes to `target-projects/<name>/` ([ADR-0015](0015-target-projects-staging.md)). One-skill-two-modes muddied two responsibilities into one prompt and risked the wrong destination on every invocation; two skills keep each prompt focused and make the destination unambiguous from the skill name. The cost is a longer skill name and a little duplicated prose between the two SKILL.md bodies — small price for clarity. The skill that ships to *projects* is the canonical `grill`, unchanged.
